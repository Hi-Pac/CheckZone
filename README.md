## نظرة عامة على المشروع

**CheckZone** هو تطبيق ويب تقدمي (PWA) مصمم للمؤسسات والشركات لتسجيل حضور وانصراف الموظفين آلياً عبر التحقق من موقعهم الجغرافي. يجب أن يكون الموظف داخل نطاق جغرافي محدد (25 متر من موقع الشركة) حتى يتمكن من تسجيل حضوره.

**آلية العمل:**
- قبل الساعة 11:00 صباحاً → يُسجَّل الطلب كـ **حضور**
- بعد الساعة 11:00 صباحاً → يُسجَّل الطلب كـ **انصراف**

### 1. استنساخ المشروع

```bash
git clone https://github.com/Hi-Pac/CheckZone.git
```

في كل ملف HTML يوجد كود مشابه لهذا:

استبدل `YOUR_SUPABASE_URL` و `YOUR_SUPABASE_ANON_KEY` بالقيم الحقيقية من لوحة تحكم Supabase.

- `index.html`
- `admin.html`
- `admin-dashboard.html`
- `salaries.html`
- `salary-settings.html`
- `deduction-settings.html`
- `penalties.html`

لا يحتاج المشروع إلى أي بناء (build)، يمكن تشغيله مباشرة عبر:
- خادم محلي بسيط مثل: `npx serve .` أو `python -m http.server`
- أو رفعه على منصة استضافة (راجع قسم [النشر والاستضافة](#النشر-والاستضافة))

## إعداد قاعدة البيانات (Supabase)

### الخطوة 1: إنشاء مشروع Supabase

1. اذهب إلى [supabase.com](https://supabase.com) وسجّل دخولاً
2. أنشئ مشروعاً جديداً واحفظ:
   - **Project URL** (مثال: `https://xxxx.supabase.co`)
   - **anon/public key** من قسم Settings > API

### الخطوة 2: تنفيذ سكريبت إنشاء الجداول

1. في لوحة تحكم Supabase اذهب إلى **SQL Editor**
2. انسخ محتوى ملف `supabase-setup.sql` والصقه في المحرر
3. اضغط **Run** لتنفيذ السكريبت

**الجداول التي سيتم إنشاؤها:**

| الجدول | الوصف |
|--------|-------|
| `employees` | بيانات الموظفين (الاسم، الراتب الساعي/الشهري) |
| `attendance` | سجلات الحضور والانصراف مع الموقع الجغرافي |
| `deduction_rules` | قواعد الخصم على التأخير |
| `penalties` | الجزاءات اليدوية للموظفين |
| `rewards` | المكافآت والبونص |
| `settings` | إعدادات النظام (بيانات المدير، موقع الشركة) |

### الخطوة 3: إضافة بيانات المدير الأولية

في **SQL Editor** نفّذ:

```sql
INSERT INTO settings (admin_username, admin_password, company_lat, company_lng, radius_meters, checkin_cutoff_hour)
VALUES ('admin', 'your_password_here', 30.8087063, 31.0030066, 25, 11);
```

> ⚠️ **تنبيه:** الكلمة السرية تُخزَّن كنص عادي حالياً. راجع [مشاكل الأمان](#مشاكل-الأمان-التي-تحتاج-معالجة).

## تغيير إعدادات الشركة والموقع

يمكن تغيير إعدادات الشركة من جدول `settings` في Supabase مباشرة أو من لوحة إعدادات الرواتب.

**الإعدادات القابلة للتعديل:**

| الإعداد | القيمة الحالية | الوصف |
|---------|----------------|-------|
| `company_lat` | `30.8087063` | خط العرض لموقع الشركة |
| `company_lng` | `31.0030066` | خط الطول لموقع الشركة |
| `radius_meters` | `25` | نطاق الحضور بالمتر |
| `checkin_cutoff_hour` | `11` | الساعة الفاصلة بين الحضور والانصراف |
| `admin_username` | `admin` | اسم مستخدم المدير |
| `admin_password` | — | كلمة سر المدير |

## طريقة الاستخدام

### للموظف (index.html)

1. افتح الموقع من هاتفك المحمول
2. اسمح للموقع بالوصول إلى موقعك الجغرافي
3. تأكد أنك داخل نطاق الشركة (25 متر)
4. اضغط على زر **تسجيل الحضور / الانصراف**
5. ستظهر رسالة تأكيد بالنجاح أو الفشل

### للمدير

1. اذهب إلى `admin.html`
2. سجّل دخولاً باسم المستخدم وكلمة السر
3. ستُوجَّه إلى `admin-dashboard.html`
4. من القائمة الجانبية يمكنك الانتقال إلى:
   - **سجلات الحضور** — عرض وفلترة وتصدير السجلات
   - **الرواتب** — حساب رواتب الموظفين
   - **الجزاءات والمكافآت** — إدارة الجزاءات والمكافآت
   - **الإعدادات** — إعداد قواعد الخصم والرواتب الأساسية

## مشاكل الأمان التي تحتاج معالجة

### 1. 🔴 كلمات السر بنص عادي (Plain-text Passwords)

**المشكلة:** كلمة سر المدير تُخزَّن في جدول `settings` كنص عادي، وتُقارَن مباشرة:

```javascript
// المشكلة في admin.html
if (data.admin_username === username && data.admin_password === password)
```

**الحل المطلوب:**
1. استخدام Supabase Edge Function للمصادقة
2. أو تفعيل Supabase Auth واستخدام نظام المصادقة المدمج
3. أو على الأقل تخزين hash للكلمة (SHA-256 أو bcrypt)

### 2. 🟡 مفاتيح API مضمّنة في الكود

**المشكلة:** مفتاح Firebase مضمّن مباشرة في `src/firebaseConfig.js`.

**الحل المطلوب:**
- استخدام متغيرات بيئة عبر build tool (Vite/Webpack)
- أو استخدام Firebase Hosting Environment Config

### 3. 🟡 سياسات RLS متساهلة

**المشكلة:** في `supabase-setup.sql` يوجد:

```sql
USING (true) WITH CHECK (true)  -- يسمح لأي شخص بالتعديل
```

**الحل المطلوب:** تقييد الوصول حسب الدور (role) أو التوكن.

### 4. 🟡 لا يوجد معدل طلبات (Rate Limiting)

**المشكلة:** يمكن لأي شخص إرسال طلبات حضور غير محدودة.

**الحل المطلوب:** إضافة Rate Limiting في Supabase Edge Functions أو Firebase Cloud Functions.

## استكشاف الأخطاء وإصلاحها

### ❌ الموقع لا يستطيع تحديد موقعي

- تأكد من أنك سمحت للمتصفح بالوصول للموقع الجغرافي
- تأكد من أن الموقع يعمل عبر HTTPS
- في Chrome: اذهب إلى `chrome://settings/content/location` وأضف الموقع في قائمة المسموح

### ❌ خطأ "أنت بعيد عن موقع الشركة"

- تأكد من أنك داخل نطاق 25 متر من الموقع المحدد
- تحقق من إحداثيات الشركة في جدول `settings` في Supabase
- GPS قد يكون غير دقيق داخل المباني، حاول الخروج

### ❌ فشل تسجيل الدخول للمدير

- تأكد من تنفيذ سكريبت `supabase-setup.sql` أولاً
- تأكد من إضافة بيانات المدير في جدول `settings`
- تحقق من بيانات Supabase URL و API Key في `admin.html`

### ❌ الصفحة لا تعمل أوفلاين

- Service Worker يحتاج إلى زيارة أولى بالإنترنت ليخزن الملفات
- تأكد من تسجيل Service Worker في المتصفح (DevTools > Application > Service Workers)
